# CSV Reader

Программа для чтения CSV-файлов и вычисления математических выражений в ячейках.

Поддерживает базовые арифметические операции (`+`, `-`, `*`, `/`) и ссылки на ячейки в формате `ИмяКолонки НомерСтроки`.

## Структура проекта
* `src/` — исходный код программы.
* `include/` — заголовочные файлы.
* `tests/` — юнит-тесты и интеграционные тесты для проверки краевых случаев.
* `Makefile` — скрипт автоматизации сборки.

## Инструкция по сборке

Для компиляции проекта потребуется компилятор `gcc` (или `clang`) и утилита `make`.

### Сборка на Linux / macOS
1. Откройте терминал и перейдите в корневую директорию проекта (туда, где лежит `Makefile`).
2. Выполните команду:
   ```bash
   make
   ```
После этого в папке проекта появится директория `build/bin/`, внутри которой будет лежать готовый исполняемый файл `csvreader`. При сборке также автоматически запустятся все тесты.

### Сборка на Windows (MSYS2 / MinGW / Cygwin)
В Windows сборку нужно производить через специальные окружения (рекомендуется MSYS2).

1. Запустите терминал вашего окружения (например, ярлык MSYS2 MinGW 64-bit). 
2. Через команду `cd` перейдите в папку с проектом.
3. Выполните команду сборки:
   ```bash
   make
   ```

### Очистка проекта
Чтобы удалить все скомпилированные объектные и исполняемые файлы (полностью удалить папку `build`), выполните:
```bash
make clean
```

## Запуск программы

Имя CSV-файла передается первым аргументом командной строки.

**Пример для Linux / macOS:**
```bash
./build/bin/csvreader input.csv
```

**Пример для Windows:**
```cmd
build\bin\csvreader.exe input.csv
```

Результат вычислений (таблица в формате CSV) выводится в стандартный поток вывода (`stdout`). Если формула содержит циклическую зависимость или деление на ноль, в ячейку выводится соответствующая ошибка (`#CYCL!` или `#DIV/0!`).

## Тестирование
Проект покрыт автоматическими тестами. Они компилируются и выполняются автоматически при каждом вызове команды `make`. 

Если сборка завершилась строкой `All tests passed!` (или аналогичной от второго тестового файла), значит программа работает корректно. 

При желании скомпилированные тесты можно запустить вручную из папки `build/bin/`:
* `./build/bin/test_mathparse.bin` — проверка математического парсера, поиска ссылок и логики вычислений.
* `./build/bin/test_output.bin` — интеграционный тест (сравнивает итоговый вывод программы в консоль с ожидаемым эталоном).
